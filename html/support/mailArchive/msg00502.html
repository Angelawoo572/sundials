I don't know where you guys got the idea that in a windows dev environment all files in a project have to be in the same directory. Of course they don't. Today I compiled CVODE 2.4 using VS 7., not a hitch. And I didn't have to move a single file. 

Same here. I compile CVODE with both Borland C++ Builder 6 and VC8, and no 
problem at all. Just set your include paths and be done, or create a 
sundials_main_header.h in your favorite directory that binds in all required 
sundials headers. Also, to assist the users in a Win dev environment you can 
always choose to compile the whole SUNDIALS code into a single library, that 
way only one lib file has to be referenced.


Slightly unrelated to that I have a small request. Would it be possible to 
expose the difference quotient implementation for the cvode band solver 
(CVBandDQJac) in some header file? I sometimes want to call this function from 
my own DQ jacobian composition algorithm and need to create my own prototype 
declaration. Also, I noticed, in the newest version of Sundials the function 
was made static and I need to hack the cvode_band.c file to access the 
function, which makes me feel kinda worried, because when something is static 
there is typically a reason for that...
