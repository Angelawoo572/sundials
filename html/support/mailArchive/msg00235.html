Hi Mukesh,

changing values in the y vector directly did not work for me either. Typically 
I keep a copy of the state variables in my y vector in linear vectors for each 
control volume and in the first loop of the CVODE f() function I transfer the 
data to those linear vectors and adjust the ranges accordingly.

Example:
2 state variables in each element, a and b:
structure of y-vector: y[0] = a0, y[1] = b0, y[2] = a1 ...

// first loop in f()
unsigned int j=0;
for (unsigned int i=0; i<n; ++i) {
        a[i] = std::max(0.0, NV_DATA_S(y)[j++]);
        b[i] = std::max(0.0, NV_DATA_S(y)[j++]);
}

// now use the vectors a and b in your code

Also, saving the states from the last call of CVODE allows a small 
optimization. Sometimes, when the values of a[i] and b[i] did not change since 
the last call of CVODE, that may save you some computational time. At least for 
me it makes a difference of about 15% since I need lots of pow() and log() 
calls in my code.
