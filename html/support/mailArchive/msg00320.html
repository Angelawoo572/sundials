sundialsTB has not received a lot of mail, so I did not bother keeping the 
release CVS branch
up-to-date. All bug fixes and updates were done only on the development branch, 
so they'll be
available in the next release only.

The cvm_rhsS1.m file was left out of the distribution tarball. It is attached 
to this message. Save
it under sundialsTB/mex/cvm.

The segfault error you see is - I assume - in the cvbx example. The reason for 
that is a buggy
implementation of CVodeMonitor which cannot work if other figures are open 
while CVodeMonitor gets
called (notice that no other examples attempt that). The new release will 
contained a fixed
CVodeMonitor function. In the meantime, to make cvbx work, there are two 
options:

a) do not do monitoring; i.e. comment out the line

   options = CVodeSetOptions(options,'MonitorFn','CVodeMonitor');

b) do not plot additional stuff; i.e. comment out the lines

   fig=figure;
   hold on;
   box

and

   figure(fig);
   plot([t_old t],[uavg_old uavg]);
   plot([t0 tout], [q q]/(tout-t0), 'r-');
   plot([tout tout], [0 q]/(tout-t0), 'r-');
