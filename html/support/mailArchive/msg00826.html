Hello Michael,

You have no choice -- changing state variables requires
re-initializing the solver.  If you change values in y and/or
yPrime externally to IDA, this has no effect on IDA, which
has a stored history internally.  Changing parameters in the
model (but not the state vectors) could be done without
restarting, but the discontinuity would still cause IDA to
perform inefficiently at each such event.

I realize that these frequent changes will make the solution
prohibitively expensive.  In fact, that is the wrong way to
solve a stochastic ODE or DAE problem.  You need to consult
literature on the solution of stochastic DEs.  (We cannot offer
any software that does this, sorry to say.)

Alan H

Hello,

I am trying to use IDA to solve a stiff 1-d problem in turbulent
combustion. Turbulence is modeled via a stochastic Linear Eddy
model. This model requires to stochastically change data in space in
prescribed time intervals.

I tried to do this change (or better re-arrangement) of data in the
solution vector u and its time derivative uPrime which I provide to
IDA in the initial call

IDAMalloc(mem, f, T0, y, yPrime, IDA_SV, RTOL, abstol);

However, that doesn't do anything and my guess is that IDA is working
internally with different data.  The data I see after

flag = IDASolve(mem, time, &tret, y, yPrime, IDA_NORMAL);

in y and yPrime are probably only a copy of IDA's internal data. Is
that right?

Question: How can I change my data/solution-values y and yPime without
re-initializing the whole solver? I have to do this re-arrangement of
data very often, so (time consuming) re-initializing is not really an
option for me.
