Am Thu, 22 Feb 2007 16:52:28 -0500 hat Radu Serban <radu@llnl.gov> geschrieben:

> The "temperature calculation" within the new RHS is nothing but an inversion 
> of
> the matrix C, correct? So you say that it's possible to efficiently invert C.
> Why would this then be any better than writing T' = C^{-1}* {original rhs} ?

No :-) For the simple heat capacity which is a typically a constant or weakly 
dependent on T, it would be absolutely the same to use the inverted capacity 
matrix. Since the internal energy would be calculated as I proposed by:

u = c(T)*rho(T)*T
   = C(T)*T

where c(T) is the specific heat capacity and rho is some mass density and the 
unknowns {y}={u}.

However, if one intends to extend the system of equations to include some more 
complex physics (I assumed that this might be necessary) the matrix C cannot be 
easily inverted. But T can be calculated, if necessary using some iteration 
scheme, from the internal energy. Also, a formulation of this kind, using the 
extensive quantities such as the internal energy directly as solution variable 
might allow a better control for energy/mass conservation (basically the 
tolerance parameters you define in CVODE for u would correspond to the accuracy 
of energy conservation).

But then again, probably doesn't make a big difference which formulation is 
used...
