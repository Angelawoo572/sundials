Title: CVodeSetMaxStep and rhs evaluation

Hello,

I am using CVode in CV_ONE_STEP_TSTOP mode. At any step (time = t), I use CVodeSetMaxStep to limit the maximum step size to h. While the actual step size is observed to be no greater than h, CVode() is evaluating the right-hand-size at time valus greater than t+h. 

Is this normal if I'm using BDF with Newton iterations and SPGMR? 

My rhs has a discontinuity at a fixed time value and I was hoping to get around it by controlling max step size. However, it doesn't work if internally the rhs is evaluated outside the time interval [t, t+h]. I can probably restart the problem at the discontinuity, but controlling max step size is much easier, especially if there are many such jumps.
