Anatoliy,

This is nothing to be concern about. Even during a successful configuration, the configure script will try to run many short programs that will (and should) fail. The test above is performed to check whether non-existent headers can be detected and how (configure creates this simple program which includes the non-existent header file "ac_nonexistent.h", expecting the program to fail in order to see how such a failure can be intercepted. Therefore, you should be concerned only if the above program did not fail!

As a matter of fact, I compared your config.log with one I generated on my laptop under cygwin and, except for obvious differences in path names, machine names, and some line numbers, they are identical.
