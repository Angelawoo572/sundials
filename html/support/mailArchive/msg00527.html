I do not disagree with either one. I also keep a local 'apps' directory with a filesystem structure that duplicates the one under /usr/local, but I cannot assume that every one of our typical users will have/do that.

Anyway, whatever will be done, there certainly will be only one -I flag required. The question is only where that flag points to and what the corresponding include directives in a user code look like. I'm willing to ignore the above concern about "typical" users. So I see the three options listed below (I was suggesting #3).

My problem with #1 and #2 is now how to organize the source (distribution) tree, given that, while building the SUNDIALS libraries, we have to compile sources/headers that have include directives like those listed below for user codes. I'd like a clean file structure and I do not want to keep, for the compilation only, different headers than those exported, nor do I want to have conditional includes in the header files (#ifdef SUNDIALS_COMPILE or something similar).

With option #3, the SUNDIALS distribution would unpack into the following 
structure:

sundials-2.2.1/
sundials-2.2.1/src/
  sundials-2.2.1/src/ida/
    sundials-2.2.1/src/ida/fcmix/
  sundials-2.2.1/src/cvode/
    sundials-2.2.1/src/cvode/fcmix
  ...
sundials-2.2.1/include/
  sundials-2.2.1/include/ida/
  sundials-2.2.1/include/cvode/
  sundials-2.2.1/include/sundials/
  ...
sundials-2.2.1/examples/
  sundials-2.2.1/examples/ida
    sundials-2.2.1/examples/ida/serial/
    sundials-2.2.1/examples/ida/parallel/
    sundials-2.2.1/examples/ida/fcmix_serial/
    sundials-2.2.1/examples/ida/fcmix_parallel/
  ...
sundials-2.2.1/doc/


What should this structure be for options #1 and #2 which you seem to 
favor? Sugestions?

--Radu



(1) All header files into ${prefix}/include/sundials

Structure of ${prefix}/include would then be:

${prefix}/include/sundials
${prefix}/include/sundials/ida.h
${prefix}/include/sundials/ida_dense.h
${prefix}/include/sundials/sundials_types.h
...

A user code would have:

#include <sundials/ida.h>
#include <sundials/ida_dense.h>
#include <sundials/sundials_types.h>

(2) Header files in a directory structure under ${prefix}/include/sundials
(In this case, the subdirectory containing the headers sundials_***.h would be 
named 'shared')

Structure of ${prefix}/include would then be:

${prefix}/include/sundials
${prefix}/include/sundials/ida.h
${prefix}/include/sundials/ida/ida_dense.h
${prefix}/include/sundials/shared/sundials_types.h

A user code would have:

#include <sundials/ida.h>
#include <sundials/ida/ida_dense.h>
#include <sundials/shared/sundials_types.h>

(3) Header files in a directory structure under ${prefix}/include
(In this case, the subdirectory containing the headers sundials_***.h would be 
named 'sundials')

Structure of ${prefix}/include would then be:

${prefix}/include
${prefix}/include/ida.h
${prefix}/include/ida/ida_dense.h
${prefix}/include/sundials/sundials_types.h

A user code would have:

#include <ida.h>
#include <ida/ida_dense.h>
#include <sundials/sundials_types.h>
