Hi Andreas,

This use of a smooth scale function to ramp in the adjustment is
a great idea.  It lets the integrator work through that spot much
more efficiently, without doing any damage to your model.
This sets a good example for others with similar situations.

It also sounds like you don't need to see the solution transient over
that 1 second, only the solution on a longer time scale.  So there is
really no need for you to switch to a local time variable tau, even if
you still get t and h values such that t + h = t.

-Alan

> Hi again,
>
> at first thanks for the suggestions!
>
> Yesterday I tried to approach the problem from a different point of view: How 
> can I modify the boundary conditions to "help" the solver without violating 
> the physical meaning/result of the simulation?
>
> Considering a simulation time of over a year, it really doesn't matter if a 
> boundary condition switches instantenously or smoothly during 1 (really 
> short) second. So I used my scaling function
>
>     double scale(double x) { return 0.5 - 0.5*cos(x*3.141592654); } // 0 < x 
> < 1 --> 0 < scale < 1
>
> once again to adjust my value smoothly from 0 to maximum and now the solver 
> slows down only to 1e-3 as opposed to 1e-9 before at the discontinuity. And 
> best of all the results look the same! I will still consider using a shifting 
> tau variable for really long time spans...
