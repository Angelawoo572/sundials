Hmm, I didn't look carefully enough through the above code snippet the 
first time around...

The problem is that you cannot use the same value 'i' to index both the sensitivity vector and elements within such a vector since, in your case Neq != Ns. In a call to the macro Ith(yS[i],j) you must have 0<=i<Ns and 0<=j<Neq.

Trying to write very compact code is not always the best idea :-) It'd be much 
more clear to write it like this:

 for(i=0; i<Ns; i++)  N_VConst(ZERO, yS[i]);
 for(i=0; i<NUM_ASTER; i++)
   for(j=0; j<6; j++) Ith(yS[7*i+j],6*i+j) = ONE;


 for(i=0; i<Neq; i++) {
   for(j=0; j<Ns; j++) printf("%4.2f  ",Ith(yS[j],i));
   printf("\n");
 }

I assume you use the macro Ith as defined in the CVODES examples. If so,
notice that its second argument runs from 1 and not 0. When i=0, you are
accessing memory outside that allocated for your vectors, hence the
SEGFAULT. Use Ith(yS[i],i+1).

Finally, if you decide to provide your own tolerances for the sensitivity variables then, in the CV_SS case, the abstolS argument to CVodeSetSensTolerances must be an array of Ns realtype values (one scalar absolute tolerance for each sensitivity vector).

I suggest however to let CVODES select the tolerances for the sensitivity 
variables but provide Ns pbar values:
        [cbar cbar cbar vbar vbar vbar mbar | cbar cbar cbar vbar vbar vbar 
mbar | ...]
where cbar is an order of magnitude for coordinates, vbar one for velocities, 
and mbar one for masses.

As a general rule, for each variable related to the state variables you must provide an array of Ns such variables for the sensitivities. For example, an N_Vector of initial conditions for the states becomes an array of Ns N_Vectors for the sensitivity initial conditions; a scalar absolute tolerance for the states becomes an array of Ns realtype absolute tolerances for the sensitivities, etc.
