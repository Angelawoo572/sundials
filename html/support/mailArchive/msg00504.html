Hi again,

to provide some insight into the steps necessary to get sundials working under 
windows and to evaluate the need and steps for improvements, here's a brief 
summary about the steps necessary to get SUNDIALS working with native compilers 
under windows:

1. create/download project files to compile SUNDIALS libs with your compiler 
(since
   project files and sundials_config.h files are often already available or 
need to
   be created only once, this step can be considered a minor issue)

   Btw, shouldn't we put the project files for VC8 and BCB6 on the FAQ page as 
well?

2. add the following include directories to your project:
   sundials/include (or whereever your own created sundials_config.h is)
   sundials/cvode/include   (may be different, depending on the solver you are 
using)
   sundials/shared/include
   sundials/nvec_serial

3. add the following library dependencies to your project:
   libSundials  (if all compiled into one lib as in my setup)

   - or -

   libCVode
   libShared
   libNVecSerial


That's all. With VC8 you don't even need to add the libraries explicitly, 
simply add the
sundials lib projects to your solution and check the appropriate dependency 
check box.

Doesn't seem too complicated to me. But while we discuss improvements, how 
about switching to a platform independent CMAKE based build system, instead of 
using autotools?
