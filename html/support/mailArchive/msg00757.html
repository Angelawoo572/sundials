I'll take a look at the code to see whether we missed any calls.

However, note that the value returned by KINGetNumFuncEvals does not take into account any function evaluations needed for the DQ Jacobian approximation. To get the number of these additional function evaluations, use the appropriate optional get function for the linear solver you're using (e.g. KINDenseGetNumFuncEvals for the DENSE linear solver).
