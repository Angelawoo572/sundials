This brings me back to my original issue:
WHen I set up 2 cells with differing initial conditions, I get the same output profile even when I leave them uncoupled!

Again these are the steps I have followed:
> 1. create cell1 - create cvode_mem ptr, init cvode for cell1 with N1 and
> state1, its state vector
> 2. create cell2 - create cvode_mem ptr, init cvode for cell2 with N2 and
> state2. its state vector
> 3. Set up for loop to simulate run
> 4. call cvode for cell1 with state1;
> 5. call cvode for cell2 with state2.

If I set up 2 different cell types, i.e N1 != N2, I just die miserably. Could this have anything to do with the way my classes are set up?

I have a basic cell class which sets up the CVODE interface. Different cell types inherit from the basic class and hence inherit the objects and methods to call CVODE.

KP,

Make sure you are not always using the same cvode_mem pointer for the two different cells (in the calls to CVode()). You do have two different cvode objects, right? If not, that would explain both "unexpected" issues: getting the same results even if the IC are different and having failures (segfaults by any chance?) when using different N1 and N2.
