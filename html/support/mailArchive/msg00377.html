Is it ok to call OtherFuncF and modify y as presented under or not

newY = NV_DATA_S(VY); 

Here you just copy pointers. Modifying the content is _not_ ok. Create a 
deep temporary copy of the data if you need to modify it in your other 
function. However, if modification of your solver variables is what you want, 
try the following:

1. create copy of calculated y values after completing a time step
2. call other func and modify temporary variables
3. calculate differences between modified and original variables
4. add those differences as constant sources/sinks to the next time step

I know this is a hack, but if you solution isn't too sensitive, this allows a 
very simple decoupling of the CVODE integration and an external 
modification/adjustment of the y variables.

Andreas

PS: If you want to call your other function to adjust unphysical values in y, 
please refer to the previous posts from the past weeks.
