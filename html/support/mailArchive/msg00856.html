I do a similar thing by way of using Delphi and its VCL as the front end but with sundials compiled with
MS C Version 6 as a DLL. Thus is it advantageous to create a DLL if you are using CBuilder. In fact, all my
solvers are in the same DLL (all with type STDCALL).

Actually, I have started to port sundials to Delphi and with some asm included as well. Except for some
selected FORTRAN compilers, Delphi generates the most optimized i386 code I have found, that is, if you
can do with i386 code, and inline asm is just too easy to mix with functions. 
I also use Delphi for my front end and use CVODE compiled using BCB2006 for the backend (using the following flags: -c -6 -O2 -Ve -X -pr -a8 -b -d -k- -vi -tWM -r -RT- -ff) through an interface that I have written in Delphi. I am quite pleased with the speed I get, but wouldn't mind even faster speeds... :)
 
Isn't porting Sundials to Delphi a big job (in particular when a new version comes out)? Also, how much of a speedup do you think you are going to get? I am curious...
