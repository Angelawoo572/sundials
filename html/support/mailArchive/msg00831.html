Assuming the SPEC file has been rpmlint'ed and put in
a form the Fedora Extra people like, I would strongly
advise putting the source RPM and .spec file on a
public server then posting on their mailing list.

If Fedora Extras doesn't like it (unlikely, as they're
borderline fanatical about this kind of package), I
strongly suggest that someone establish a YUM server
that hosts either this package specifically or this
kind of package in general. Programmers who are also
system administrators are rarer than programmers who
can bribe the admins to install a pre-rolled package.

As it stands, I'm having a few problems with it. On
trying to do an rpmbuild, I get the following error:

error: Installed (but unpackaged) file(s) found:
   /usr/bin/sundials-config

I'd also like to suggest that the SPEC file produce
different RPMs for the different subsections (CVODE,
CVODES, IDA, KINSOL), that libraries are in a -libs
and headers are in a -devel. This seems to be the
general convention, as much as I happen to dislike
large numbers of packages.
