From Radu Serban <radu@llnl.gov>

John,

It would quite helpful if there were some kind of wiki or webpage where
people could document their results at integrating the various sparse
direct solvers with the various solvers in SUNDIALS. In many cases we
could eliminate duplicated effort, I think.


Such a wiki page exists. See my reply to Andrey.

Rather than adding a sparse direct solver into SUNDIALS, perhaps an
option would be to allow SUNDIALS to detect solvers installed on the
local system and build support for the ones that it finds?  With UMFPACK
and KLU already available in Ubuntu and Fedora package repositories (and
SuperLU on Ubuntu only), this would be an efficient way forward.

I never suggested I was going to write a sparse direct linear solver for SUNDIALS (I wouldn't know how - that's not my cup of tea) but simply providing interfaces to one or more. All the ones that were mentioned so far are possible candidates.


--Radu
