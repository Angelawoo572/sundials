Hi,

John W. Eaton wrote:
> ... In your sources, you can keep the directory structure
> however you like and use
> 
>   #if defined (SUNDIALS_LIBRARY)
>   #include "other.h"
>   #else
>   #include <sundials/other.h>
>   #endif
> 

Oh, no.

Radu Serban wrote:
>
> I'll keep thinking about a cleaner way...
>


(1) Yes, the SUNDIALS source code should use ONE line of code for each header:

    #include <sundials/sundials_config.h>
    #include <ida/ida.h>


(2) Maybe it is helpful to move headers like ida.h to it's
    subdirectory ida/


(3) In source tree the subdirectory structure is fine:

    <builddir>/src/sundials/sundials_config.h
                            *.c
                            *.h
                            ...

    <builddir>/src/ida/ida.h
                            *.c
                            *.h
                            ...

(4) For build process you can use ONE include path

    CPPFLAGS  = -I<builddir>/src


(5) Target structure:

    <prefix>/include/sundials/sundials_config.h
                              ...
                     ida/ida.h
                              ...
    <prefix>/lib/libsundials*.so (*.a)

(6) User's point of view:

    SUNDIALS_HOME   = <prefix>
    CPPFLAGS        = -I$(SUNDIALS_HOME)/include
    LDFLAGS         = -L$(SUNDIALS_HOME)/lib

What do you mean?

/André
