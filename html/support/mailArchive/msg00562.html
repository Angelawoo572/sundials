Dear All,

KINSOL sometimes throws a floating point overflow error at this point in the
code (~line 1565 in kinsol.c):

          if (omega == ZERO){
        /**** This causes floating point overflow if fnorm>>fnormtol:
*****/
        omega = MIN(omega_min*EXP(MAX(ZERO,(fnorm/fnormtol)-ONE)),
omega_max);

Now, obviously, this doesn't seem to be a problem for most people since this
issue doesn't come up in the archives or FAQ. But if fnorm >> fnormtol then
EXP(form/fnormtol) will blow up. This, I presume, will happen quite
frequently at the beginning of a problem. However, since omega has an upper
bound, one can avoid calling EXP with large arguments by calculating the max
value of fnorm/ftol that will result in omega >= omega_max.

This value is fnorm/ftol >= ln(omega_max/omega_min)+1. However, we can also
avoid making a call to ln by merely approximating it with a Taylor exansion
around a convenient point. I used 10 here, and since the linearisation of LN
is always >= LN itself, this is an inexpensive check against a large value
being fed to EXP.

So I add the following code:

                /*Change to FP-friendly version:
                    I use a Taylor approximation around x=10 */
                  if (fnorm/fnormtol> 0.1*omega_max/omega_min+2.3 )
                          omega = omega_max;
                  else omega =
MIN(omega_min*EXP(MAX(ZERO,(fnorm/fnormtol)-ONE)), omega_max);
          }
