[sundials-users] Building and using SUNDIALS on Windows with Microsoft Visual Studio 2005

From "Paolo Greppi" <paolo.greppi@tiscali.it>
Date Thu, 30 Nov 2006 17:10:07 +0100 
Hi all,

many ideas have been posted on this list on how to suild SUNDIALS on windows
with Microsoft Visual Studio 2005.
I am posting one more solution because it is based on an approach I could
not find in any of these previous posts, i.e.:
- have ./configure do the job under cygwin and avoid tweaking any file
- avoid .vcproj file; just start make under cygwin and have it launch
Microsoft Visual Studio 2005's CL.EXE
- minimizing moving around of files (i.e. not altering the 

The procedure is (in this case for IDA 2.5.0):
1) Open the "Visual Studio 2005 Command Prompt" (after you install should be
in Start->Programs->Visual C++ 2005 Express Edition->Visual Studio Tools);
this will set the environment set for using Microsoft Visual Studio 2005 x86
tools
2) From within the same CMD shell, start your cygwin installation's bash
shell, something like: "C:\Program Files\cygwin\bin\bash --login -i"; the
environment setings will be inherited and now cl and lib should be in the
path for bash
3) Extract the SUNDIALS tar.gz archive choosing a location such to avoid any
space in the path, easy thing is to put it in / or /cygdrive/c which is
cygwin's path to C: "cd /cygdrive/c; tar xvzf ida-2.5.0.tar.gz; cd
ida-2.5.0/"
4) start configure "./configure CC=cl LD=lib"
5) make it "make"
6) Copy file sundials_config.h from ...\ida-2.5.0\src\sundials to
...\ida-2.5.0\include\sundials (the ellipsis indicates the full path to
where you extracted the tar.gz archive)
7) copy libraries ...\ida-2.5.0\src\ida\.libs\libsundials_ida.lib and
...\ida-2.5.0\src\nvec_ser\.libs\libsundials_nvecserial.lib to wherever you
like

To use the generated libraries in a project:
1) start new Win32 Console Application project, tick the box "empty project"
2) add existing item, ida-2.5.0\examples\ida\serial\idadenx.c
3) in Project->Properties->Confguration Properties->C/C++->General set
Additional Include Directories to "...\ida-2.5.0\include"
4) in ...->Linker->Input enter libsundials_ida.lib
libsundials_nvecserial.lib in additional dependencies

Paolo
