On 10/14/05, Radu Serban <serban1@llnl.gov> wrote:
> Hi Raju,
>
> The -ffloat-store is a gcc/g77 that enforces exact IEEE conformance on some 
> machines (such as x86 and m68k GNU systems).
> We add this flag by default to CFLAGS if configure decides that the C 
> compiler is GNU C and to FFLAGS is configure
> thinks the compiler is "GNU Fortran 77".
>
> In some older version of the code which was computing the unit roundoff 
> internally (now we grab it from float.h) having
> this flag was crucial so we added it in the Makefiles in the source 
> directories (it was not necessary in the Makefiles
> for the SUNDIALS test examples). Later on, some users had floating point 
> problems in their own codes that were resolved
> by using this flag so we decided to add -ffloat-store to the Makefiles for 
> the SUNDIALS test examples just to make users
> aware that it may be needed.
>
> For some odd reason, configures identifies the later Absoft f90 compilers as 
> "GNU Fortran 77". Other people have the
> same issue (see 
> http://lists.gnu.org/archive/html/autoconf/2000-11/msg00040.html).
>
> Without the autoconf source files, the only fix I can suggest for you right 
> now is to edit the configure script and
> comment out the lines
> FFLAGS="-ffloat-store"
> and
> FFLAGS="${FFLAGS} -ffloat-store"
> and then run configure again.
>


I removed the -ffloat-store option from both the above lines and used
the following configure command.

/home/software/compileHere/sundials/configure CC=gcc F77=f90 \
--with-mpiroot=/home/software/compiledLibs/mpich2_1.0.1_gcc_4.1.0_absoft_8.0 \
2>&1 | tee configure.log

make 2>&1 | tee make.log
make install 2>&1 | tee install.log
make examples 2>&1 | tee examples.log
make clean 2>&1 | tee clean.log
make examples_clean 2>&1 | tee examples_clean.log

With just the above configure options, there entire suite compiles
just fine. I guess then the whole problems are due to the fact that

The absoft compiler is being recognized as GNU F77 compiler. Due to
this, -ffloat-store option is getting added to the FFLAGS. But since
this option is not present in absoft, the configure command is giving
all sorts of errors.

I wonder if there is any better solution. But for the moment I was
able to compile fine (both the code & examples). I will see how the
tests work out and get back to the list if there were more problems.
