Hi Greg,

The initial step size will be different in the two cases. unless provided by the user, the initial step size is computed internally based,
among other things, on tout. T! his calculation is done only at the first step, so T1 is used in case 1 and T1*pow(TMULT,NOUT-1) is used in
case 2.
You can obtain the value of h0 using CVodeGetActualInitStep. You'll see that h0 is larger for the second case (probably too large, so there
will be a couple more of error test failures)

Try to set the same initial step size with CVodeSetInitStep for the two cases and compare stats.

--Radu
