From Radu Serban <serban1@llnl.gov>

The configure script was not able to determine the name-mangling scheme (in order to call a Fortran subroutine from C, we first need to decide what we name that Fortran subroutine so that we can refer to it from C).

As you noticed with gfortran, the configure script can find the name-mangling scheme automatically for most compilers, but not all. This is why we provide the alternative of manually specifying it, using the configure options --with-f77underscore and --with-f77case. Check the documentation for the Fortran compiler to find out the case and the number of underscores appended to the Fortran routine names.

A quick online search gives this:

-----
Using "man f90" on a Linux platform with Absoft f90 installed reveals
that in addition to -B108, you should be using  "-YEXT_NAMES=LCS"
For example with the source code:

        call foo
        end

And the command line  "f90  -B108 -YEXT_NAMES=LCS j.f", yields
the confirmation from the linker:

        j.o: In function `main':
        j.o(.text+0x29): undefined reference to `foo_'

That the name mapping you desire is achieved.
-----

So with the above two flags to absoft's f90 the name-mangling scheme is apparently "one underscore + lower case". I don't have access to this compiler to check, but I assume that using only the -B108 flag will result in the scheme "one underscore + upper case"

Try first to add one or both flags to f90:
% configure --with-fflags=-B108 ....
or
% configure --with-fflags="-B108 -YEXT_NAMES=LCS" ....
and see if configure will be able to find the name-mangling scheme by itself.

If this does not work, try
% configure --with-fflags="-B108 -YEXT_NAMES=LCS" --with-f77underscore=one 
--with-f77case=lower ....
or
% configure --with-fflags=-B108 --with-f77underscore=one --with-f77case=upper 
....

--Radu
