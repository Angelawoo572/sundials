Andreas Nicolai wrote: 
Hi there,

Slightly unrelated to that I have a small request. Would it be possible to expose the difference quotient implementation for the cvode band solver (CVBandDQJac) in some header file? I sometimes want to call this function from my own DQ jacobian composition algorithm and need to create my own prototype declaration. Also, I noticed, in the newest version of Sundials the function was made static and I need to hack the cvode_band.c file to access the function, which makes me feel kinda worried, because when something is static there is typically a reason for that...

Bye,
Andreas


Andreas,

That would require exposing a bit too much of the CVODE internals for my taste. We'd need to also export the implementation header file cvode_band_impl.h which contains the CVBandMemRec structure and I'd rather not do that (especially since we'd have to do the same for all other linear solver modules).

Is there any reason why you don't simply duplicate the DQ Jacobian computation in CVBandDQJac into your code? As noted in the documentation, anything that CVBandDQJac uses by accessing directly the CVODE memory structure can be obtained by different means (current stepsize, error weights, etc. can be obtained by calling CVodeGet* functions). Moreover, with your own version, you could also tune the DQ computation to your application (I don't know what that is but the banded DQ Jacobian evaluation could be made more efficient if, for example, you were using it for a semi-discretized PDE which would have a banded Jacobian but with many zero upper and lower diagonals. CVBandDQJac would then explicitly compute all these zeros.)
