Hi Einat,

First of all, please direct all SUNDIALS-related questions to the sundials-users mailing list (to which you can subscribe by filling up the form at http://www.llnl.gov/casc/sundials/support/support.html) so that the whole project here at LLNL, as well as all the members of that mailing list, can see them and have a chance to answer.

Regarding your question: indeed, there is a bug in the configuration files used to build sundialsTB using autotools. I forgot to also link into the MEX files the serial nvector library when parallel support is enabled (which happens if you have in your path an MPI implementation that supports the MPI-2 specification). If you do not care about parallel support, simply add the --disable-mpi option to your configure line.

The simplest solution however, is to install sundialsTB using the Matlab script install_STB.m provided in the directory sundialsTB/ of the sundials distro. Simply start Matlab in that directory and run install_STB. Note that the checks for enabling parallel support in sundialsTB when using the install_STB.m script are a little bit stricter: you must have both LAM MPI and MPITB available (tested through appropriate environment variables).

Hope that this helps,
--Radu
