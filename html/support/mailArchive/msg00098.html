On Tue, 2005-07-19 at 17:26 -0700, Keith Eric Grant wrote:
> I've 
> started a background (aka evening/weekend) project of a Python 
> implementation of parts of SUNDIALS. At this point, I've done the 
> generic NVector routines and implemented and tested about 2/3 of the 
> routines for a scalar NVector implementation. 

Thanks for the update! I'd be interested to be able to use the
capabilities of sundials from Python.

A simple extension wrapper of the sundials C codebase seems to be only
the first step of large effort that would be needed. 

Another issue I soon discovered was the callback mechanisms for residual
and Jacobian evaluation, currently pointers to C functions with specific
argument signatures that include NVectors; calling back into a Python
method would be a preferred option for casual scripting language use of
sundials.

Finally, the NVector Python operations that you're working on would seem
to be a critical fundamental building block that could be helpful in
brokering between NVector and Numeric arrays, or numarrays, or whatever
they end up being in the long run.

However, to meet my more pressing and immediate needs, I'll take the
path of least resistance and just implement my small DAE system in C
using the existing infrastructure, for which I'm grateful.

In the long run, though, I'd be interested in having a Python interface
available to sundials powerful capabilities, so please keep us posted
about any progress!

Thanks,
Phil S.
