Hi Mike,

Cygwin is, in my opinion, the easiest way to build SUNDIALS under
Widows. Except for a few tricks needed to compile shared libraries,
everything works just as under any other *NIX.

Here's what happens in your case:

1) Yes, you do have to specify CC and CFLAGS if your compiler is not in
the standard location (without them, SUNDIALS would probably build
using gcc). 

2) You can safely ignore the warnings from configure re. standard
libraries. There was a small bug in the configuration files that I've
fixed for the next release

3) You can safely ignore the cygwin-related warnings from "make
install". I don't understand why libtool even issues that warning,
given you are not trying to build shared libraries...

4) The example that you show is a Fortran program. Can you compile any
C example? For this, try to go in (for example)
sundials/cvode/examples_ser and do "make install" there.

Assuming that only the Fortran examples don't compile: Did configure
find a Fortran compiler? What did the report printed at the end of the
configuration say about which examples will be configured? If you do
have a Fortran compiler available, you may have to use F77 and FFLAGS
on the configure line (same as with CC anf CFLAGS). If you do not have
one, just pass the option --disable-f77 (I'm not 100% sure about the
exact name, check it with --help) to configure.

Hope that this helps.
--Radu
