Bug report 1: Shared library have wrong extensions on MacOSX 10.4.7 x86

Unfortunately we do not have access to this platform and therefore cannot verify this (although I know others have successfully built SUNDIALS on MacOS - but maybe not with shared library enabled and maybe not this particular version?).
Does anyone else have experience with this?


Bug report 2: Incorrect include in header files

I closed this bug with "InvalidBug" resolution and the following comments:

The SUNDIALS version reported (2.1.1) is probably not the one that was used:

In v2.1.1 the header files are all exported to the same directory (e.g.
/usr/local/include)

In v.2.2.0, the header files are indeed exported to subdirectories, and it is
the user's responsibility to add all required preprocessor flags (e.g.
-I/usr/local/include -I/usr/local/include/cvode -I/usr/local/include/sundials).
This can be seen in the Makefiles generated by configure in the example
directories (e.g. see sundials/cvode/examples_ser/Makefile).

Note that the next release will include header files with the mechanism
suggested by the reporter.


Hope that this helps,
--Radu
