Hi Alan and Michael,

using Sundials in some other toolkit that doesn't support C linking directly (e.g Delphi or BCB if you think it is too slow) is probably best done using a wrapper DLL that exposes the important function interfaces, like Michael is using it right now. If the wrapper DLL is written properly, it wouldn't need much modifications when new sundials versions come out. So porting sundials to Delphi would IMHO not be the perfect solution when considering future updates. Btw, if someone has created such a DLL, would it be possible to share it with the user group?

For my part, I link the sundials solver directly into my GUI tool as "internal solver", which is unfortunately not so fast, but ok for 1D simulations. For 2D simulations I also provide an external command line based solver that I compile with gcc (and which also runs on our linux cluster).

For people still looking at GUI toolkits to create GUIs for Sundials solver based applications, I would recommend testing Qt - it allows you to use any reasonable fast C/C++ compiler.
