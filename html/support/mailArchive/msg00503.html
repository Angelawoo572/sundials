I don't know where you guys got the idea that in a windows dev environment all files in a project have to be in the same directory. Of course they don't. Today I compiled CVODE 2.4 using VS 7., not a hitch. And I didn't have to move a single file.


Same here. I compile CVODE with both Borland C++ Builder 6 and VC8, and no problem at all. Just set your include paths and be done, or create a sundials_main_header.h in your favorite directory that binds in all required sundials headers. Also, to assist the users in a Win dev environment you can always choose to compile the whole SUNDIALS code into a single library, that way only one lib file has to be referenced.
