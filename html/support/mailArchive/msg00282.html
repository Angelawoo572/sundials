Hi Mukesh,

> b) My simulation runs for 2500 time steps and after that it gives
> segmentation fault. Since in each time step, solver accesses the same
> memory structure I can't think of a reason why it gives segmentation error
> after 2500 time steps instead of the first one itself.

Try to run your code with Valgrind. Your segmentation fault after some time 
looks like an access violation that only matters after "something" in your 
code happens. This could be triggered by some floating point 
overflow/underflow that somehow screws up your memory addressing (do you use 
pointer arithmetics in your code?).

Also, maybe you accidently stay within your own allocated memory for some time 
before you actually access something else?
