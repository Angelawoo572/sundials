Re: Undefined value from CVodeGetCurrentStep

From Radu Serban <serban1@llnl.gov>
Date Tue, 17 Jan 2006 09:06:59 -0800
Andreas Nicolai wrote: 
Hi again,

I call CVodeGetCurrentStep during the calculation of my f() function. However, this yields undefined results when the f() function is called the first time in the CVode() function.

Is there a way to find out whether the result of CVodeGetCurrentStep is valid or not from within the f() function?

Andreas


Andreas,

CVodeGetCurrentStep returns the value of the step size to be ATTEMPTED on the next step (next_h). This may very well be different from the step size that ends up being used (hu, returned by CVodeGetLastStep). As such, next_h is set and should be queried/used only AFTER a successful step. Therefore, it does not make much sense to call CVodeGetCurrentStep from within your right hand side function f().

I don't know how you use the step size inside f(), but it would probably make more sense to use the value of the step size used on the last successful step, hu. Note that hu=0.0 as long as nst=0 (nst being the value returned by CVodeGetNumSteps).


--Radu

P.S. I will however initialize next_h to 0 in CVodeCreate, but only so that we do not have uninitialized variables in the code...
