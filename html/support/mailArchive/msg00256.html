Andreas Nicolai wrote:

Why not use the same interface system that makes Sundials already very flexible? We could just have another interface function like:

    CVodeSetErrorHandler(cvodemem, error_handler)

where error_handler() is some user defined function like

    void error_handler(int error_code, const char * msg, void * data);

Whenever such an error_handler function is set, instead of writing error messages directly with fprintf() all those calls are simply replaced with an sprintf into a buffer and a call to error_handler.

Would that work while keeping the required code modifications small?


I think this is a fine idea... this is essentially what I did, brute force. This allows users to handle the message themselves, which is what a library should really do. (And would of course make the upgrade path easier for those like me who have already taken matters into their own hands and made upgrades with future versions and bug fixes of CVODE a bit more tedious for ourselves...)
