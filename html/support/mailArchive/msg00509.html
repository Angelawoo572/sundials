OK. Below is what I'm thinking of doing. Please let me know if you have any comments and/or suggestions. Would this make things easier or more difficult for Windows users? I suspect it wouldn't make a difference.

Once the changes are made and released, I hope some of you could provide project files for VC7, VC8, BCB6, etc. so that I can make them available on the SUNDIALS website. Andreas, did I hear you volunteer? :-)

Anything I missed?

--Radu



1. Keep the directory layout for the exported header files as it is right now.

That is, the main solver and nvector header files in the top directory <sundials_install_path>/include, all other solver specific header files in <sundials_install_path>/include/<solver>, and all shared header files in <sundials_install_path>/include/sundials

2. For CVODES, combine cvodes.h and cvodea.h into a single cvodes.h.

3. Change all #include directives throughout to use the path relative to <sundials_install_path>/include and hence require (at most) one preprocessor -I flag.

That is #include "cvode/cvode_dense.h" and so on.

4. Rearrange the SUNDIALS source tree into a more standard layout.

The new structure of the SUNDIALS source tree <sundials_source_path> is given below ([*] are directories). With this structure, the only problem I can foresee is getting errors at "make install" if using the same build and source tree (which is the current default if prefix is not specified at configure time). I'm actually thinking of also modifying the configure script so that it defaults to the standard /usr/local prefix and implicitly encourage users to install to a location different than the source tree.

<sun_source_path>

  [include]
      cvode.h
      cvodes.h
      ida.h
      kinsol.h
      [cvode]
      [cvodes]
      [ida]
      [kinsol]
      [sundials]
      [nvec_ser]
      [nvec_par]

  [src]
      [cvode]
           [fcmix]
      [cvodes]
      [ida]
          [fcmix]
      [kinsol]
          [fcmix]
      [sundials]
      [nvec_ser]
      [nvec_par]

  [examples_ser]
      [cvode]
          [fcmix]
      [cvodes]
      [ida]
          [fcmix]
      [kinsol]
          [fcmix]

  [examples_par]
      [cvode]
          [fcmix]
      [cvodes]
      [ida]
      [fcmix]
      [kinsol]
          [fcmix]

  [doc]
      [cvode]
      [cvodes]
      [ida]
      [kinsol]
