John W. Eaton wrote: 
On 14-Oct-2005, Radu Serban wrote:

| We'll think about including the Autoconf input files in the next
| release.

Why would you not include them?  They are the real source files that
are needed if one wants to make modifications and contribute
improvements.

jwe


A few reasons would be:

1) They include code for the development version of SUNDIALS.
2) We have a separate configure.ac that is used to generate the configure script for the distribution tarballs that only include the serial vector module (i.e., no MPI support).
3) We'd have to support those too :-) We were hoping that users will use the configure script as a black-box.

However, I did not say we will not include them in the next release.

--Radu
