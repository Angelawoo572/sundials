I just answered a part of my questions myself:

I tried to run the CVSpgmr with a dense preconditioner matrix, this
mistake caused the increased CPU time...

Now I'm only using the J_{i,i} elements and the preconditioned CVSpgmr
is now faster than CVDese. :-)
