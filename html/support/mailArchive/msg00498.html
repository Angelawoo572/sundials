Hi all,

I would prefer the POSIX-style of header inclusion:

    #include "ida.h"
    #include "ida/ida_band.h"
              ^^^^

    or

    #include "sundials/ida.h"
    #include "sundials/ida/ida_band.h"
              ^^^^^^^^^^^^^

    and

    CPPFLAGS = -I<sundials-path>/include

instead of the current solution


    #include "ida.h"
    #include "ida_band.h"

    and

    CPPFLAGS = -I<sundials-path>/include -I<sundials-path>/include/ida
                                                                  ^^^^

The reason is very simple: If the SUNDIALS package is installed
in a standard location no additional include paths are necessary.

Herbert Sauro wrote:
> ...
> I think it would be much better to be consistent on all systems, and
> to do things in a way that would not require adding a bunch of -I
> flags to the compiler command line.  But if you really must have
> different directory layouts on different systems, then how about at
> least giving us the option of installing things in multiple
> directories, and not having to specify multiple -I options on the
> command line?  

/André
