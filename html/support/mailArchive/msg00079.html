Hi Herbert,

your question may be related to a problem I had some time ago (it may also be completely unrelated, but I write it anyway :-)

Inside the f() function called from CVODE you get the y-vector and ydot-vector as N_Vectors. It is important to note, that the underlying memory area (where the actual vector data is stored) may be changing in subsequent calls of the f() function. So obtaining the y or ydot memory location and "remembering" it somewhere separate as double pointer to the memory block will not work (I tried and spent half a week looking for the bug :-)


Actually the N_Vectors themselves change between different calls to the RHS function, but that's less important. As Andreas noted, do not attempt to save and reuse these addresses between subsequent calls.

So, now I always use the macro NV_DATA_S to access data of the y and yend vectors and with modern compilers there's almost no performance penalty for that.

Indeed, the cost of the NV_DATA_S (or NV_DATA_P) macro is only one explicit cast operation and two structure (->) operations.
