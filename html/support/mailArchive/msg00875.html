[sundials-users] Changing internal time step size?

From Gregory Mason <gregbrickmason@yahoo.com>
Date Mon, 12 Feb 2007 11:15:51 -0800 (PST)

Hi There
 I am solving a large system of stiff ODEs.
 The forcing values to the ODEs (these are time series which are input on the right hand side of the ODEs) are measured at different time intervals. That means the forcing data I have can be at seconds, minutes, hours, days interval. Now, in most of the existing models that solve similar  problem, explicit aprroximate techniques have been used oflate. So with the change in forcing data interval, they are able to run the model at time steps equal to forcing data interval. 
 Now while using CVODE (SPGMR, newton-krylov scheme), obviously depending on the stability, accuracy requirements etc., the time steps change here adaptively. That is well and good when the forcing is applied at high time resolution (smaller time interval) thus resulting in accurate calculation of state variables. But what if, the time interval of forcing data is large (say a month). In this case, any way the faster state variable with time scale (<<< month) wont' be a true value but merely an exact solution of a problem fed by inexact forcing. So in this case I will be more interested in average variations of some of the relatively slower state variables only.
 When I use CVODE, i dont' get larger internal time steps even when I am using a monthly forcing data. And that is expected as forcing at a large time interval just translates to a constant forcing for that particular interval. 
 All this is ok, untill I want to run a very long simulation with forcing data at large time intervals. In this case I am willing to trade accuracy of faster variables with a faster simulation.
I tried achieving this with relaxed tolerance. But still, even when I go from a "seconds" forcing input interval to a "month", by relaxing tolerance - average increase in internal time step size increases marginally only (1.2 to 1.5 times).
 Any thoughts over this. I am hoping that I can run the accurate simulation (with smaller forcing intervals)  and the approximate (with large forcing intervals) when needed using the same CVODE setup. And that I can get faster speedup proportional to (month/second) ratio.
 
Regards
Greg
