I wrote this query into llnl and got an answer from Radu Serban, thought I'd 
share it with the mailing list.
 
> I am writing to ask you a question concerning the use of CVODE and is related 
> to work we are conducting for the DOE/GTL program. We are currently moving 
> from LSODA over to CVODE because CVODE is much faster. However we have 
> encourtered one problem which is to do with making discrete changes to 
> parameter values in between consecutive integration runs. Luckily LSODA 
> provides a means to reset the integrator when a parameter has changed and 
> allows us to start a new integration without having to create a new 
> integrator from scratch. We couldn't find the equivalent functionlity in 
> CVODE, we noticed the function CVReInit an/d was wondering if this could 
> serve as a means to reinitialise the integrator after we've changed a 
> parameter I presume the alternative is to free the current integrator and 
> create a new one?

        > 
        > Any advice on this matter would be gratefully received. 
        > 
        > thanks in advance 
        > 
        > Herbert Sauro 
        > 

        Dear Herbert, 

        Yes, CVodeReInit is exactly what you need. See the examples cvdemd, 
cvdemk, cvkxb, and pvkxb where this is used.
