Hi Andreas and Keith, 

what is happening here is that for Na+ = Cl+ < 6 the solid phase is not
anymore stable and disappears.
This is a transition to a different region of the phase space with a
different variance. Some people call it a catastrophe. 
It is a catastrophe for the numerical implementation for sure, because it
would require the equation / variable count to be updated etc. 

We can work around this and avoid the troubles in may ways; Andreas' own
solution is OK if you write a custom solver, Keith's solution is brilliant
and also has a physical meaning.
If we rewrite:
        ln K_NaCl - ln a_Na+ - ln a_Cl- = 0
        K_NaCl = a_x * a_y / (1/b)

then we can interpret 1/b as the activity of the solid, that while the
equilibium is active is assumed unitary and incorporated in the K_NaCl.
Outside the validity region of the equilibrium Keith proposes to assume a
metastable pseudo-solid in equilibrium with ions. The activity of the
metastable pseudo-solid is less than one..

As such this example matches a variety of situations, if you want to see
examples of similar "tweaks" in the area of vapor-liquid equiliria you can
have a look at:

"Effective utilization of equations of state for thermodynamic properties in
process simulation" (pagine 182-186) P. M. Mathias, J. F. Boston, S.
Watanasiri AIChE Journal Volume 30, Issue 2 (March 1984)

Thanks you for proposing such application-oriented examples !

Andreas Nicolai wrote:

> 
> Mathematically this corresponds to:
> 
> x + z - x_0 = 0
> y + z - y_0 = 0
> C - ln a_x - ln a_y = 0
> 
> with a_x = a_x(x,y)  and a_y = a_y(x,y) and  C = const, and x,y,z >= 0
> 
> Solving this problem with Kinsol is very easy provided that _enough_ 
> NaCl  is present. Since negative mol numbers do not make sense the 
> solution has  to be restricted to positive values. Now when the amount 
> of NaCl available  is too small, equation 3 can never be fulfilled and 
> thus KinSol (or any  other iteration algorithm) cannot find a solution.
> 

Andreas,

Consider your 3rd equation, derived from

a_x * a_y <= C

Replace this by

b * a_x * a_y = C,

Then for b >= 1, the equation should always be solvable.

Write

Q = (ln C - ln a_x - ln a_y - ln b)^2 + alpha*(b-1)^2,

with alpha a relatively small parameter.

You want to minimize Q with respect to ln a_x, ln a_y, and b

For the first two, differentiating and setting the result equal to zero just
returns your original 3rd equation with the additional "-ln b" term

Minimizing Q with respect to b gives an additional equation,

(ln C - ln a_x - ln a_y - ln b) - alpha*b(b-1) = 0


Now, if your 3rd equation is solvable to begin with, what I've written
should solve with b=1. If that isn't the case, it tries to find a solution
with (b-1)^2 a minimum.
