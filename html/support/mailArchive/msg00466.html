Hi everybody,

I am installing a program (BioNetGen2: network.c, run_network.c) that uses a lot of include files and libraries from the SUNDIAL package, really only for calling CVODE. However, it uses an older version of SUNDIAL. I have downloaded the newest SUNDIAL and there seems to some changes in header file names and libraries, including their location.

I have found most of the new names and locations of the include files. For example, there is now a directory in /include, called /sundials and this contains some of the old/new headers. I am not 100 % sure of which old SUNDIAL version BioNetGen uses but there are some definite differences in names and locations of the headers.

Here is a list of the include files that BioNetGen is looking for (presumed to be in /include):

#include "sundialstypes.h"
#include "cvode.h"
#include "nvector_serial.h"
#include "cvdense.h"
#include "dense.h"
#include "cvspgmr.h"
#include "spgmr.h"

I have matched all of these up I think to the new names and locations...

Here is a list of the libraries the linker is looking for:

SUNDIALS_LIBS    = -lsundials_cvode -lsundials_nvecserial -lsundials_shared

It is the last one (shared) that I cannot find anymore.

Has anyone had to map the names and locations of include files and libraries as SUNDIAL has been slowly modified? Does anyone have this mapping?
