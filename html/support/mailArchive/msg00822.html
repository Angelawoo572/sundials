Hi James,

We do not provide the ebuild script for SUNDIALS, so this question is better addressed to whoever contributed it to Gentoo and in whatever forum such issues are dealt with. I am not familiar with ebuild but:
1) it's obvious that it tries to install in /usr where, I assume, you have no write rights.
2) it seems to me that simply replacing /usr in the src_install() section with a directory for which you do have the right permissions should fix the install problem.

--Radu

P.S. By the way, that ebuild script seems extremely basic. Maybe you should consider using the configure script distributed with SUNDIALS which provides so many more options.
