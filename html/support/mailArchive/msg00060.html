Hello Woo-Chang,

I'm compiling the sundials lib under windows using the BCB6 compiler. I have 
simply created a sundials_config.h by copying it from the template and 
adjusting the defines that are normally modified by the installation script.

I simply looked up the header files mentioned manually (you can just type the header 
files into any source code of yours as for instance #include <float.h> and use 
the IDE to look up the source code file - if it is found just change the #undef to a 
#define in the appropriate line of the config.h. I ended up with the following 
defines:

#undef F77_FUNC
#undef HAVE_DLFCN_H
#define HAVE_FLOAT_H
#undef HAVE_INTTYPES_H
#undef HAVE_LIBM
#define HAVE_MATH_H
#define HAVE_MEMORY_H
#define HAVE_STDINT_H
#define HAVE_STDLIB_H
#undef HAVE_STRINGS_H
#define HAVE_STRING_H
#define HAVE_SYS_STAT_H
#define HAVE_SYS_TYPES_H
#undef HAVE_UNISTD_H
#undef PACKAGE_BUGREPORT
#define PACKAGE_NAME "CVODES 2.2.2"
#define PACKAGE_STRING "CVODES library version 2.2.2"
#undef PACKAGE_TARNAME
#undef PACKAGE_VERSION
#undef SIZEOF_DOUBLE
#undef SIZEOF_FLOAT
#undef SIZEOF_INT
#undef SIZEOF_LONG_DOUBLE
#undef SIZEOF_LONG_INT
#define STDC_HEADERS
#undef SUNDIALS_CASE_LOWER
#undef SUNDIALS_CASE_UPPER
#define SUNDIALS_DOUBLE_PRECISION
#undef SUNDIALS_EXTENDED_PRECISION
#undef SUNDIALS_SINGLE_PRECISION
#undef SUNDIALS_UNDERSCORE_NONE
#undef SUNDIALS_UNDERSCORE_ONE
#undef SUNDIALS_UNDERSCORE_TWO
#define SUNDIALS_USE_GENERIC_MATH

That works fine on my system. Simply add the path to your modified 
sundials_config.h as additional include dir to your VC++ project options and 
you'll be fine.
