Re: CVodeSetMaxStep and rhs evaluation

From Alan Hindmarsh <alanh@llnl.gov>
Date Mon, 11 Jul 2005 14:32:44 -0700 (PDT) 
The TSTOP options (normal or one-step) are the way to treat the problem
of a discontinuity at a known time, not the HMAX option.  HMAX is not
enforced universally.  Also, using both options is inefficient.

If you are using CV_ONE_STEP_TSTOP, then your RHS function should
never get called with t beyond TSTOP (except by roundoff).  If it is,
we should look for a bug.  I say "except by roundoff" because, on a
step that would otherwise overtake TSTOP, the value of t is set by
lines like:
    hprime = tstop - tn;
    t = tn + hprime;
Roundoff can make this t go beyond tstop in the last bit.
You should make your RHS function safe for this possibility.  If
necessary, have it use a separate t value set by something like:
   tt = MIN(t,tstop);
(if t is increasing), but only if t - tstop > 0 is found to be at the
level of roundoff in t.

If you are seeing t overtake tstop by roundoff, then maybe we should
put in a fix in CVODE that adjusts t values to avoid this.  We'll
consider that.

-Alan Hindmarsh
