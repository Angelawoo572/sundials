On 15-Jun-2006, Radu Serban wrote:

| I left out a bit of the story. Rearranging the source tree is actually needed 
in order to compile the SUNDIALS libraries given the changes 
| to the include directives (since at that time the header files are not 
exported yet). The only two other alternatives I can think of are:
| (a) condition the include directives on whether we compile the solver 
libraries or an example, e.g.
| #ifdef SUNDIALS_COMPILE
| #include <sundials_nvector.h>
| #else
| #include <sundials/sundials_nvector.h>
| #endif
| (b) rearrange only the header files within their include directories in the 
current layout of the source tree. But that would imply creating 
| some awkward-looking subdirectories, such as 
<sundials_source>/shared/include/sundials and 
<sundials_source>/cvode/include/cvode.

Or, if you use a single directory for installed header files
($includedir/sundials), then

  * Your sources don't need to change.  They can continue to use

      #include "ida.h"
 
    For the build, you can add whatever -I flags are necessary to
    find these files.  It doesn't matter if that requires several -I
    options, it is just the build step.

  * Users can have

      #include <sundials/ida.h>

    and only have to add -I$includedir to their compiler flags (or
    add no extra flags if $includedir is searched by default).

So dumping all the installed header files for SUNDIALS into one
directory seems like the simplest solution that causes the least
amount of pain.  Is there any reason not to do this?

jwe
