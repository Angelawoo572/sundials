As to the DAE system you have, I strongly suggest that you make it
into an ODE system as follows:
(1) Assuming A1 is nonzero, the first equation can be divided by A1
    to give x_dot = ... explicitly.
(2) The second and third equation involve y_dot and z_dot, with a
2x2 matrix whose elements are (reading rowwise) B1+B3, B3, C3, C1+C3.
Assuming that this matrix is nonsingular, it is easly to write out its
inverse explicitly, and with that, you can write explicit equations
    y_dot = ...  and  z_dot = ...
After doing that, use CVODE on the resulting ODE system.  Of course,
if those coefficients are constant, compute the inverse elements only
once, save them, and reuse them as needed.  CVODE is generally less
expensive (and easier to use) than IDA, for systems that are equivalent.

If I am wrong about the nonzero/nonsingularity assumptions, then
indeed IDA is the right code to use.  Then you need to define a vector
of 3 residual functions, which are to be equal to zero for the
solution.  For example, the third residual function would be
  Res3 = C1*z_dot +C2*(x+y^2+z^2) + C3*(y_dot+ z_dot) - F2(y,z)

In any case, this is certainly not a problem that should be run
in parallel, if it is really only a system of 3 equations.

-Alan H.
