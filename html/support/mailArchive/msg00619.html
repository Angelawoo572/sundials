[sundials-users] Typo in kinsol documentation?

From "Cayle Sharrock" <cayle@chemeng.uct.ac.za>
Date Fri, 18 Aug 2006 17:09:40 +0200 
According to the Kinsol documentation, specifying constraints in
KinSetConstraints is as follows (pg48):

 Arguments: If constraints[i] is
        0.0 then no constraint is imposed on ui.
        1.0 then ui will be constrained to be ui > 0.0.
         -1.0 then ui will be constrained to be ui < 0.0.
          2.0 then ui will be constrained to be ui >= 0.0.
         -2.0 then ui will be constrained to be ui <= 0.0.

Should the ones and two not be swapped around, so that -1 represents <= and
-2 represents < and so on?
This would make the documentation consistent with the demo code (kindenx) as
well as my experience of trying to use 2/-2 and setting initial conditions
to the constrained value, which causes an IllegalInput error.
