Hello Andreas,

It is important to find out which of the two cases you have: stiffness
or high-frequency oscillation.  If it is (only) stiffness, CVODE
should be able to handle the problem, with the right inputs.  With
Newton iteration, what is your choice of linear system solver?
Possibly that choice is leaving something out of the Jacobian in the
3-equation case, causing it to fail to converge.  If you are using
preconditioned GMRES, this means that maybe your preconditioner is
leaving too much out.  In this case you should be seeing high
convergence failure counts (nncfails from CVodeGetNonlinSolvStats).

On the other hand, if it is the high-frequency oscillation case, a
radically different approach may be needed.  You might try the
stability-limit detection option (CVodeSetStabLimDet), which limits
the BDF order if it senses a slowly damped high-frequency oscillatory
mode.  If you do that, find out if that algorithm actually does any
order reductions (CVodeGetNumStabLimOrderReds), to confirm the
problem.  If that doesn't work, you might try temporarily relaxing
tolerances until the oscillatory mode (hopefully) damps out.  If this
mode is undamped, you might think about reformulating the equations so
as to get rid of that mode.

-Alan
