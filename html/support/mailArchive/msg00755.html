Hi Aria,

I would like to comment on your question as well. Adrian is right that just 
because SUNDIALS is written in C, there shouldn't be any major problems of 
interfacing with Fortran code, particularly because the interfaces are plain C 
and function arguments are either single values or array, both which can be 
passed without any problems from Fortran to C functions and vice versa. There's 
a lot of information on the web on how to do this for the various platforms, 
but you should be able to get this running in short time. Also, the SUNDIALS 
solvers come already with interface functions for Fortran which make this 
really easy.

But I would also like to comment on your other question...

>>  I use odepack, dassl, and daspk in Fortran, but now I
>> realized the next generation is in C. Should we change
>> Fortran with C?

I used to program in Fortran 77 and 90 for a while and in my opinion fortran is 
a very powerful language for mathematical computing, provided that your program 
does _not_ aim at providing a lot of flexibility. Whenever you try to program 
complex, flexible systems (and that starts with user editable input files, 
possible some input as XML from database etc., but continues with flexible 
memory management depending on user options or even interaction with the user 
during the runtime of your program) you will find that Fortran is not the best 
language to use. Despite recent additions to the language (not yet standard, 
though), like OOP in Fortran95, it is by design limited... providing the 
simplicity of use that you mention has its prize when it comes to larger, 
complex and most of all flexible systems.

Btw, the Sundials libraries are really really flexible... and that would have 
been difficult to achieve with Fortran.

But for you the question will really be the scope of your applications: if you 
are only solving well defined problems, and you don't want to provide flexible 
interfaces for users (including parsing input files, dynamically managing all 
the memory in your program etc.) you should be perfectly fine with Fortran - 
even save a lot of time. But if you want to "step up", mainly in terms of 
usability and applicability of your programs by users that may not be 
programmers, C/C++ might be a good alternative to look at.

Mind, that's just my oppinion :-)
