must N_VSpace return an exact result?

From Kevin Long <krlong@sandia.gov>
Date Fri, 09 Dec 2005 16:30:59 -0800 
Hi,

I'm working on writing code to wrap Trilinos abstract vectors and solvers in 
the SUNDIALS N_Vector and solver interfaces. Most of the N_Vector functions 
are simple to implement with Trilinos objects. However, it's not easy for me 
to implement N_VSpace with arbitrary Trilinos vectors. So my question is: is 
it necessary to get an exact measure of memory required?

I noticed that in IDAMalloc, if the nvspace function pointer is null,  lrw and 
liw are simply set to zero and the code proceeds happily along. After that, 
the only access to workspace information is through IDAGetWorkSpace, which 
seems to be mainly for diagnostic output. 

Am I correct in understanding that N_VSpace is used for diagnostic information 
only, and that things will not crash and burn if I simply provide a rough 
estimate of vector storage requirements?

Thanks,
