Hi all,

IMHO this is the crucial point for both, Windows and UNIX/Linux:

Andreas Nicolai wrote:
> ...
> 
> 2. add the following include directories to your project:
>    sundials/include (or whereever your own created sundials_config.h is)
>    sundials/cvode/include   (may be different, depending on the solver
> you are using)
>    sundials/shared/include
>    sundials/nvec_serial
> 

I know many packages that use exactly one include path
(and library path), e.g.:

POSIX (UNIX, Linux, ...):

    /usr/include/stdio.h
    /usr/include/iso/stdio_iso.h

    #include <stdio.h>
    #include <iso/stdio_iso.h>

    CPPFLAGS (ONE include path needed):  -I/usr/include


X11:

    /usr/X/include/Xol/Primitive.h
    /usr/X/include/X11/Intrinsic.h

    #include <Xol/Primitive.h>
    #include <X11/Intrinsic.h>

    CPPFLAGS (ONE include path needed):  -I/usr/X/include


GTK:

    /usr/gtk-1.2/include/gdk/gdk.h
    /usr/gtk-1.2/include/gtk/gtkwidget.h

    #include <gdk/gdk.h>
    #include <gtk/gtkwidget.h>

    CPPFLAGS (ONE include path needed):  -I/usr/gtk-1.2/include


I would propose the same style for SUNDIALS - it
is not really necessary but common practice.
For makefiles or within an IDE you have to configure
only one include path.

/André
