C++ wrapper for IDA

From Paolo Greppi <paolo.greppi@tiscali.it>
Date Fri, 08 Jul 2005 21:50:08 +0200 
I agree that C is better for your project, but as a library user I like C++
for manipulating the high complexity objects that occur in my project.
So I thought I could share this C++ wrapper for IDA, actually just a
prototype for dense IDA.
I have applied it to the irobx example shipped with the product to
cross-check.

One problem I have found is that in IDAMalloc I have to pass a pointer to a
static function res. But I would like to pass it the member res of a
particular instance of a subclass of DAE_objective_function !
The way to work around this I found is via a global pointer to function:
DAE_objective_function *IDA_DAEsolver_ofp_
The static, global res function calls the appropriate member res of a
particular instance of a subclass of DAE_objective_function via this
pointer.
This is VERY dirty and makes the IDA class a little bit non reentrant. If
anybody has a better solution, I would be glad.
 
Another peculiarity is that I wanted to use std::vector<double> as dense
serial vector storage, so I believe I have written a first-approx NVector
wrapper for it.
This in turn allows to interface with gmm++ which I like for matrix and
vector manipulations.
