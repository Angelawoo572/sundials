The predictor polynomial is built up with the following conditions
1) \omega_p(t_{n-i}) = y_{n-i}, i=1,...,q
2) \dot \omega_p(t_{n-i}) = \dot y_{n-1},

which I now understand. But, when I look up the description in "The PVODE and IDA Algorithms" by Alan Hindmarsh I do not see how condition 2)
is satisfied. The predictor polynomial is constructed from a set of calculated
points y_{n-i}. I can not find the point where \dot y_{n-1} is plugged in.

What you need to realize is that the both the predictor and corrector polynomials are encoded in the Nordsieck history array. As such, at the beginning of a new step, the Nordsieck array encodes the corrector polynomial from the previous step which becomes the predictor polynomial for the current step. In other words, suppose we've just successfully reached time t_{n-1}. This means that the Nordsieck array has just been corrected to contain the corrector polynomial at that step which satisfies \dot y_{n-1} = \dot \omega_{c, n-1} (t_{n-1}) (because this is how we actually define the BDF on the previous step - see step (3) in my previous email). But \omega_{c,n-1} \equiv \omega_{p,n} and therefore \dot \omega_{p,n} (t_{n-i}) = \dot y_{n-1}.
