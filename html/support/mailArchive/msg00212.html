Hi Alan,

I wrote my own version of Weight Function and set this function using
IDASetEwtFn() after calling IDAMalloc(). Now do I need to set the values of
RTOL and ATOL vectors in the program?

Actually, I tried to put uninitialized version of ATOL(N_Vector) and RTOL in
IDAMalloc() because I don't need them in the program. But this is causing
error because some of the uninitialized values of ATOL are negative. May you
please explain why I need to set the RTOL and ATOL even after setting my own
version of Weight function?

Thanks in advance,

Vinayak 

Hi Vinayak,

No, you do not need to set rtol and atol if you provide your own weight function. However, calling IDASetEwtFn only "declares" the user provided function but that function will be used only if you "enable" it by passing IDA_WF as the itol argument to IDAMalloc (the 6th argument). I suspect you use IDA_SV, correct?

If itol=IDA_WF then the rtol and atol arguments are ignored, so you could just pass 0.0 for rtol and NULL for atol.

We do not have an IDA example which illustrates this but you can take a look at the cvdenx_uw CVODE example (or cvsdenx_uw CVODES example) which use this feature (the mechanism is similar in all 3 solvers).
