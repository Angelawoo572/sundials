Re: [sundials-users] Arbitrary precision for sundials solvers?

From Radu Serban <serban1@llnl.gov>

Kai Hartmann wrote: 
Hi all,

I'm currently testing the cvode solver on a specific problem and it
seems that the extended precision is not sufficient.

Do you provide an option or include an arbitrary precision arithmetic?
Does it make sense to do so when using an ODE solver (runtime and/or
other issues)?

Kai


Hi Kai,

I suppose you have good reasons to believe that extended precision is not enough. What tolerance levels does your problem need? I would give serious thought before concluding that higher precision levels are needed.

That being said, no, we do not provide an arbitrary precision arithmetic option. However, it should be possible to use an arbitrary precision library with SUNDIALS by redefining realtype (in sundials_types.h) and, depending on the particular library used, possibly the math functions in sundials_math.{h,c}. Also, you'll have to deal somehow with the error message formats defined in the implementation header files (such as cvode_impl.h).

--Radu
