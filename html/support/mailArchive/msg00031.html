Jordan,
You are not the first to be a bit confused on how CVODE works in NORMAL vs. ONE_STEP mode. I thought I'll give a lengthier explanation of this first, before showing you how to properly code your restarting logic.
--Radu



First of all, you need to understand the distinction between tout and tret: tout is the time at which you require output (in NORMAL mode) or the time towards which the integrator should proceed (in ONE_STEP mode). tret is the time at which Cvode returns at report the solution y. In other words, in NORMAL mode tret=tout, while in ONE_STEP mode, tret<tout.

Maybe the figure below (use fixed fonts for it to make sense) will help. Suppose you want the solution at the time tout=T marked with 'tout' below and you initialize the solver at the time 't0'. Whether you call CVode in ONE_STEP or NORMAL mode, the integrator will proceed towards tout by stepping to t1, t2, ..., t6, using steps of sizes h1, h2, ...., h6. Note that, except for a freaky coincidence, tout itself is never an internal time for the integrator (i.e., CVODE does not step exactly to tout -- well, unless you set tstop=tout using CVodeSetStopTime, but that's another story). 



t0____t1_____t2_______t3________t4_________________t5_________tout________t6_________
 |     |      |        |         |                  |                      |
 |     |      |        |         |                  |                      |
   h1    h2      h3         h4            h5                   h6



(A) Now, say you want to get the solution at tout using NORMAL mode. Your 
code will look something like this

--------------------------------------------------------------------------
realtype tret, tout;
......
tout = T;
flag = CVode(cvode_mem, tout, y, &tret, CV_NORMAL);
printf("tout = %e  tret = %e   y = .... \n",tout,tret,y);
--------------------------------------------------------------------------

and your output will be
tout = T  tret = T   y = y(T)

Internally, CVODE has actually reached t6, but since you requested output at tout, it will find the solution at tout by interpolating its internal state using y(t6), y(t5), y(t4),... -- the number of past values used depends on what the current method order is -- and return tret=tout and y=y(tout).

(B) Now, if you were to call CVode in ONE_STEP mode, you should do something 
like:

--------------------------------------------------------------------------
realtype tret, tout;
....
tout = T;
do {
  flag = CVode(cvode_mem, tout, y, &tret, CV_ONE_STEP);
  printf("tout = %e  tret = %e\n  y = ....",tout,tret,y);
} while (tret<tout);
CVodeGetDky(cvode_mem, tout, 0, y);
printf("y = ...",y);
--------------------------------------------------------------------------

and your output will look like
tout = T  tret = t1   y = y(t1)
tout = T  tret = t2   y = y(t2)
tout = T  tret = t3   y = y(t3)
tout = T  tret = t4   y = y(t4)
tout = T  tret = t5   y = y(t5)
tout = T  tret = t6   y = y(t6)
y = y(T)


Note the way to extract the solution at exactly tout in the case you use 
ONE_STEP mode (using CVodeGetDky).


Now you should be able to code correctly your logic for reinitializing 
CVode at disctime.

(A) NORMAL mode:

--------------------------------------------------------------------------
void *cvode_mem;
N_Vector y;
realtype t_init=0.0, tout, tret, disctime=2.0;

// initialize y = y(t_init);

cvode_mem = CvodeCreate(...);
flag = CVodeMalloc(cvode_mem,...,t_init,y,...);

for(tout=0.1; tout<4.0; tout += .1) {
  flag = CVode(cvode_mem, tout, y, &tret, CV_NORMAL);
  // here tret=tout and y=y(tout)
  if (disctime == tout) {
     N_VScale(4., y, y);
     flag = CVodeReInit(cvode_mem,...,disctime,y,...);
  }
}
--------------------------------------------------------------------------

Alternatively, if you only want the solution at time 4.0, after a rescaling 
happens at time 2.0, you could just do:

--------------------------------------------------------------------------
void *cvode_mem;
N_Vector y;
realtype t_init=0.0, tout=4.0, tret, disctime=2.0;

// initialize y = y(t_init);

cvode_mem = CvodeCreate(...);
flag = CVodeMalloc(cvode_mem,...,t_init,y,...);

flag = CVode(cvode_mem, disctime, y, &tret, CV_NORMAL);
// here tret=disctime and y=y(disctime)
N_VScale(4., y, y);
flag = CVodeReInit(cvode_mem,...,disctime,y,...);
flag = CVode(cvode_mem, tout, y, &tret, CV_NORMAL);
// here tret=tout and y=y(tout)
--------------------------------------------------------------------------


(B) ONE_STEP mode:

--------------------------------------------------------------------------
void *cvode_mem;
N_Vector y;
realtype t_init=0.0, tout, tret, disctime=2.0;

// initialize y = y(t_init);

cvode_mem = CvodeCreate(...);
flag = CVodeMalloc(cvode_mem,...,t_init,y,...);

tout = 4.0;
do {
  flag = CVode(cvode_mem, tout, y, &tret, CV_ONE_STEP);
  // here tret != tout  and y=y(tret)
  if (tret >= disctime) {
     CVodeGetDky(cvode_mem, disctime, 0, y);
     // here y=y(disctime)
     N_VScale(4., y, y);
     flag = CVodeReInit(cvode_mem,...,disctime,y,...);
  }
} while(tret<tout);
--------------------------------------------------------------------------


I hope this helps -- Radu

Jordan:
A couple of points:
1) To reinitialize CVODE, use CVodeReInit instead of CVodeMalloc, since


the problem size remains the same. You do not need to reallocate memory


for cvode_mem...
2) After reinitialization, the integrator will restart from order 1. Although we do provide a function to set the initial step size (CVodeSetInitStep), keep in mind that the step size before the reinitialization is based on whatever the current method order was at that time and could therefore be too large. Better let CVODE select it.
--Radu

As far as I know, in order to do this you have to reinitialize the solver. The vector y returned by your call to CVode(mem, t, y, &treturn, itask) is not the state vector internally maintained by the


solver. The solver may have proceeded past your integration time t,

and

is giving you an interpolation back to the time requested, so its internal state may already be into the future. You want to change

the

internal state of the solver, and this requires initialization. You


might want to get the current internal step size of the solver and use that when you reinitialize.

JPH
