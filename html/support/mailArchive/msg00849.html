Hi there,

just tested the CMake system with the Intel Compiler on Windows. Works like a 
charm, too :-)

One thing I noticed, if you also have Visual Studio 2005 installed (which is 
the default IDE for the Intel Compilers), CMakeSetup will first detect the 
cl.exe (Visual C compiler) instead of icl.exe. The solution for this is to 
simply set the environment variable CC to icl

set CC=icl

in the command prompt that spawns from the "Build Environment for IA-32 
applications" link created by the Intel installer. Mixed compiling with the 
Fortran compiler works smoothly, too (you may need to manually call the 
ifortvars.bat batch file from your command line).

On a side note, right now I have 4 compilers on my computer: BCB6, Visual 
Studio 2005, Intel Compiler and MinGW32-GCC... and I can build sundials with 
either of them, without having problems with interfering compilers and 
libraries.
