Alan Hindmarsh schrieb:
> Yes, gamma will change from step to step, because of changes in
> both stepsize and order.  In all cases, it is important that the
> current value be used, where it can be.  In the preconditioner
> solve, this means that if your solution algorithm uses gamma
> explicitly, then use the current value, which may differ from
> what it was in the last setup call.  However, the most common
> preconditioner choice uses an LU method, and then the preconditioner
> solve function does not use gamma, but just does a back-solve
> based on preprocessing done in the setup function.
> 
> -Alan H.
