Well, not really. Your KDevelop setup is probably not the reason for the linker error since the linker finds the libraries.

Anyway, here are the instructions for KDevelop. If you have a default sundials built, the libraries are built as static libraries and after a 'make install' you can find those libraries and their headers in the corresponding subdies in the sundials dir.

If you have an Automake based project, simple open your executable project in the upper pane of the automake manager, and right click the executable target in the lower pane of the AM and open the target "options". In the configuration settings go to the libraries tab and add the sundials libraries.

NOTE: The order in which you add the libraries is important!

If the libraries are found and linked (you don't get a "can't find library xxx" warning), a wrong order of libraries is most likely the cause of your linker errors.

Good luck,
Andreas
Hello,

I haven't used KDevelop but, on most systems, the order matters because the linker won't re-scan a library once it is done with it. The primary rule is that if library a "uses" library b, then library a will appear on the command line before library b.

The correct order for CVODES is then (see also the Makefiles in the example_*** directories):
-lsundials_cvode -lsundials_nvecserial -lsundials_shared

In other words, the NVECTOR library uses symbols defined in the SHARED library and the CVODES library uses symbol from the other two.
