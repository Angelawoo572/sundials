Hi,

ok, seems we have a conflict of interests here :-) Essentially there are two 
requirements to be met:

Windows - Header and Lib files under the same base/install directory

../sundials
../sundials/include
../sundials/include/<solverXYZ>
../sundials/libs

(instead of putting the final libraries under libs I would propose a directory 
structure depending on the build system/IDE, but that's not the issue right now)


Unix/Linux (Debian and others if build as system wide lib) - Header under 
system wide include dir, libs under libs

/usr/include/sundials
/usr/include/sundials/<solverXYZ>
/usr/libs

or

/usr/local/include/sundials
/usr/local/include/sundials/<solverXYZ>
/usr/local/libs


Considering that under Windows a "make install" is typically not done or 
even necessary, wouldn't it make sense to use the Unix/Linux layout for the autotools 
based configuration?

Under Windows an installation of the library is typically just "unzip in some user 
defined dir, open the project for your IDE, compile and link your projects to whatever 
dir you have everything in".

I would vote for a unix/linux based build system which puts the sundials 
headers in the hierarchy proposed by Radu under /usr/include/sundials... or 
/usr/local/include/sundials...
