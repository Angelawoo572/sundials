David Goodwin wrote:
I am trying to port an application to the latest release of CVODES, and it seems that N_VNewVectorArray_Serial is no longer present. Is there an alternative way to create the array for the call to CVodeSensMalloc?

Thanks,
Dave Goodwin



Hi Dave,

Indeed, we have moved to a "cloning" mechanism for creating new vectors and/or arrays of vectors. For the serial NVECTOR module, there are now only two functions which create an N_Vector from scratch: N_VNew_Serial and N_VNewEmpty_Serial (the latter does not allocate space for the actual data array so that an existing one could be attached). All other vectors (including the arrays of N_Vector needed for sensitivity analysis) can then be created through "cloning" using an N_VClone***_Serial function. These functions take as one of their arguments a previously created N_Vector which will be used as a template for cloning.

You should therefore use N_VCloneVectorArray_Serial and pass it as a template the vector of initial conditions for the states for example. See any of the CVODES forward sensitivity example programs (cvsfwd***.c).
