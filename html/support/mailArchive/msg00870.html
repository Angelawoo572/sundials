Hi Andreas,

> using Sundials in some other toolkit that doesn't support C 
> linking directly (e.g Delphi or BCB if you think it is too 
> slow) is probably best done using a wrapper DLL that exposes 
> the important function interfaces, like Michael is using it 
> right now. If the wrapper DLL is written properly, it 
> wouldn't need much modifications when new sundials versions 
> come out. So porting sundials to Delphi would IMHO not be the 
> perfect solution when considering future updates. Btw, if 
> someone has created such a DLL, would it be possible to share 
> it with the user group?
> 
> For my part, I link the sundials solver directly into my GUI 
> tool as "internal solver", which is unfortunately not so 
> fast, but ok for 1D simulations. For 2D simulations I also 
> provide an external command line based solver that I compile 
> with gcc (and which also runs on our linux cluster).

What do you exactly mean by 'link[ing] the sundials solver directly into my
GUI tool as "internal solver"'? For my application, I have compiled the
files from the Sundials library that I need to use CVODE within my GUI. More
precisely, I have compiled some .c files using Borland C++ Builder to get
some .obj files that I then statically link to my application. I then get
access to CVODE through a small interface that I have written in Borland
Delphi, interface that exposes the functions that I need (CVODECreate,
CVODEMalloc, CVODEFree, etc.). From there, I have created another interface
for CVODE (that is common to the other integrators that I use within my
application), so that I can use any integrator in the same 'way'. I am more
than happy to share my code with anyone who is interested, but keep in mind
that it has been kind of taylored for my own needs...

> For people still looking at GUI toolkits to create GUIs for 
> Sundials solver based applications, I would recommend testing 
> Qt - it allows you to use any reasonable fast C/C++ compiler.
> 
> PS: Alan, I'll get those BCB performance numbers to you shortly.

Thanks, I look forward to seeing them...
