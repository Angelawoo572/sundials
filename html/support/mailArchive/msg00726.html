Hi,
I am trying to set up CVODE to solve DFN states.  I know the model is fairly accurate because with a timestep of about 0.01 ms I can run the model with just basic Y = Y +ydot*dtand get decent results.

The model consists of about 16 states and corresponding first degree equations;  The states ARE dependent on one another.

The chief parameter of interest is Vm (state [0]), the membrane voltage.  I get continuous "mxsteps steps taken before reaching tout" error and the output sort of oscillates between -200 and +20 instead of going from -80 to about +20, falling sharply and then combing back to around -80.  These cells are autorhythmic and the curves repeat automatically.

If someone can see where I am erring please please help!

This is the function f called by cvode to calculate the RHS::

static int f(realtype t, N_Vector y, N_Vector ydot, void *f_data)
{
    double ydiff[16],uinit[16];
    int z;

//put y vector into a temperory state vector
    for ( z = 0; z < DFN_STATES; z++)
        uinit[z]=NV_Ith_S(y,z);
    //cellptr->updateState(uinit);

//call cell to compute ydot.
    cellptr->cvodeCompute(uinit);
    cellptr->getDiff(ydiff);

//compute ydot vector used by cvode
    for ( z = 0; z < DFN_STATES; z++)
        NV_Ith_S(ydot,z) = ydiff[z];

    return(0);
}
