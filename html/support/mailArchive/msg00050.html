Slow performance when solution is not changing

From Andreas Nicolai <Andreas.Nicolai@gmx.net>
Date Sun, 15 May 2005 22:08:29 -0400 
Hi there,

I have a problem with one of my simulations. I calculate a 2D system with 3 
balance equations. If I run the solver with 2 balance equations it runs very 
fast. However, when I add another balance equation, having boundary conditions 
the same as initial conditions for this quantity, I get a very very poor 
performance. The calculated state variable of the third balance equation does 
not change after the first second or so. But after that initial change, the 
solver just gets very very slow. When I change the boundary conditions, so that 
the field changes, the performance improves drastically.

I tried varying the tolerances but neighter increasing nor decreasing brought 
any improvement. I also tried to restrict the solver to first order BDF scheme, 
but that also didn't help much. When I monitor the automatically chosen order, 
the solver sticks to first or at max second order while having very very small 
time steps.

From other users of CVODE I heard similar reports, as soon as there is not much 
change in a state variable the solver gets very slow.

I suspect a stability problem, but then again when I fix the first order scheme 
the solver really shouldn't have any stability issues.

Is there any trick that helps here or what should I try to do to get the solver 
to run faster?
