Hi Mzolisi,

if I understand you right, you indent to port CVode (or other parts of the 
sundials suite) to Delphi, that is Object Pascal. This is IMHO a waste of time 
and energy. I believe what you really need is to write a Delphi (Object Pascal) 
interface to access the CVode functions  from a system/programming language you 
are familiar with.

I'm currently developing with Borland C++ Builder and sometimes we mix Delphi 
and C++ in projects (to keep older modules working). So mixed language 
programming isn't really a problem. My suggestion is, that you google the web 
for finding the corresponding Object Pascal declarations for the CVode 
functions, create yourself a CVode-Interface unit and just link the C++ object 
libraries to your project. That should work fine since Borland Delphi and 
Borland C++ object code is binary compatible.

If you need assistance to get the Sundials suite compiled (using the free 
borland C++ compiler), just mail me... I'll create the required makefiles for 
you.
